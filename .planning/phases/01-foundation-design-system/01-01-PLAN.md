  <action>
  1. **Replace src/app/globals.css** with the full WildWood design system. Use the RESEARCH.md globals.css code example as the base. It includes:
     - `@import "tailwindcss"` and `@import "tw-animate-css"`
     - `:root` block with admin (light) theme: cream background (#FAF7F2 as OKLCH), wood primary (#8B6B3D), orange accent (#C94E0A), lime for success (#7AB648), sidebar dark wood
     - `.pos-theme` class block with POS (dark) theme: dark background (#1F1F1F), white text, wood/orange accents
     - `@theme inline` block mapping ALL CSS variables to Tailwind utilities: colors (background, foreground, primary, accent, wildwood-orange, wildwood-bois, wildwood-lime, wildwood-dark, wildwood-cream, sidebar-*, chart-*, banner-*), fonts (--font-sans, --font-display), border radii
     - Banner CSS variables (--banner-bg, --banner-text)

     IMPORTANT pitfalls to avoid:
     - Do NOT use raw space-separated color values (old Tailwind v3 pattern). Always use oklch() wrapping.
     - Do NOT create a tailwind.config.js file. Tailwind v4 is CSS-first.
     - Verify the OKLCH values render close to the hex originals. Use the values from RESEARCH.md.

  2. **Update src/app/layout.tsx** to configure fonts:
     - Import `Inter` and `Oswald` from `next/font/google`
     - Inter: subsets ['latin'], display 'swap', variable '--font-inter'
     - Oswald: subsets ['latin'], display 'swap', variable '--font-oswald' (it's a variable font, no weight specification needed)
     - Apply both font variables on `<html>` element: `className={`${inter.variable} ${oswald.variable}`}`
     - Set `lang="fr"` on `<html>` (French interface)
     - Add `className="font-sans antialiased"` on `<body>`
     - Set metadata: title "WildWood ERP", description "WildWood Beach Fitness & Resort - Systeme de gestion", robots "noindex, nofollow"
     - Keep layout minimal for now (no AuthProvider yet -- that comes in Plan 03)

  3. **Customize src/components/ui/button.tsx** to add POS variants:
     - Add variant `pos`: `"bg-[oklch(0.45_0.12_55)] text-white hover:bg-[oklch(0.52_0.12_55)] border border-[oklch(0.35_0.10_55)] shadow-md"` (wood tones)
     - Add variant `pos-accent`: `"bg-[oklch(0.55_0.18_35)] text-white hover:bg-[oklch(0.60_0.18_35)] shadow-md"` (orange)
     - Add size `pos`: `"min-h-[80px] min-w-[120px] px-4 py-3 text-lg font-bold rounded-xl"`
     - These meet the 120x80px minimum touch target requirement for POS on tablet.

     NOTE: Prefer using the Tailwind utility classes that map to CSS variables (e.g., `bg-wildwood-bois`, `bg-wildwood-orange`) over hardcoded OKLCH values in the button variants, since the @theme inline block already maps these. If the shadcn button cva setup supports this cleanly, use the mapped classes. Otherwise fall back to direct OKLCH values.
