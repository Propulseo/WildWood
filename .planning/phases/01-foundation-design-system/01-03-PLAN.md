  <action>
  Create directory `src/lib/contexts/` and file `auth-context.tsx`.

  Use the RESEARCH.md AuthContext code example as the implementation. Key requirements:

  1. Must be a Client Component (`'use client'` at top)
  2. Type Role = 'admin' | 'staff' (import from types.ts or define locally -- prefer import from types.ts since Role is already defined there)
  3. AuthContextType interface with:
     - `role: Role | null`
     - `login: (role: Role) => void`
     - `logout: () => void`
     - `toggleRole: () => void` -- switches admin<->staff instantly without returning to login
     - `isAuthenticated: boolean`
  4. AuthProvider component:
     - `useState<Role | null>(null)` for role
     - `useState(false)` for isHydrated flag
     - First useEffect: read `localStorage.getItem('wildwood-role')` on mount, set role if valid, set isHydrated=true
     - Second useEffect: sync role to localStorage on change (only when isHydrated is true to avoid clearing on mount)
     - `login(newRole)`: set role
     - `logout()`: set role to null (this clears localStorage via the sync effect)
     - `toggleRole()`: flip admin<->staff
     - Return `null` until isHydrated is true (prevents hydration mismatch -- server renders nothing, client renders nothing until state loaded, then both render consistently)
  5. `useAuth()` hook: reads context, throws if used outside AuthProvider

  CRITICAL: The isHydrated guard is essential. Without it, the server renders with role=null but the client hydrates with role from localStorage, causing a React hydration mismatch error. See RESEARCH.md Pitfall 1.
  </action>

  **1. Create src/components/prototype-banner.tsx:**

  A fixed banner at the top of every page, non-dismissable, showing "PROTOTYPE -- Donnees fictives".

  ```typescript
  export function PrototypeBanner() {
    return (
      <div className="fixed top-0 left-0 right-0 z-50 bg-banner-bg text-banner-text text-center py-1 text-xs font-medium tracking-wide">
        PROTOTYPE — Donnees fictives
      </div>
    )
  }
  ```

  Design decisions (Claude's discretion per CONTEXT.md):
  - Use the `bg-banner-bg` and `text-banner-text` Tailwind classes which map to CSS variables defined in globals.css (warm amber background, dark brown text)
  - `z-50` ensures it's always on top
  - `fixed` positioning so it stays visible even if a page scrolls
  - `text-xs` and `py-1` make it thin and unobtrusive but always visible
  - `tracking-wide` for readability
  - Use the em-dash character "—" not double hyphens

  This is a Server Component (no 'use client' needed) since it has no interactivity.
